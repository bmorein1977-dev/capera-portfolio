What’s inside

801_cleanup_orphan_data.sql

Removes assessor allocations and assessments that point to users who no longer exist (this is why “Emma Wilson / Alex Thompson” appear).

802_add_is_archived_column.sql

Adds users.is_archived boolean DEFAULT false (+ index).

server.routes.userArchive.patch.ts

POST /api/users/:id/archive → sets is_archived=true

POST /api/users/:id/reactivate → sets is_archived=false

server.routes.assessorCandidates.filter.patch.ts

Replaces your candidates query to exclude archived users:

WHERE aa.assessor_id = $1 AND u.is_archived IS NOT TRUE


client.UserArchiveButtons.patch.tsx

React buttons for Archive / Reactivate on the user detail panel.

client.FilterOutArchived.note.md

One-line reminder to add WHERE is_archived IS NOT TRUE to all listings.
